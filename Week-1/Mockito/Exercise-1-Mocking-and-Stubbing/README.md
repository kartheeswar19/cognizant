# Exercise 1: Mocking and Stubbing

This exercise demonstrates how Mockito helps test code that depends on external services.

- Mocking: Creating a fake object that stands in for a real dependency.
- Stubbing: Defining what the mock should return when a method is called.
- Purpose of Mockito: Simplify unit testing by isolating code from external dependencies.
- Expected output: The service returns the stubbed value when the mocked dependency is used.
